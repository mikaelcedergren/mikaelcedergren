# mikaelcedergren.com

Mikael Cedergren's personal portfolio + blog, rebuilt as an **Angular 22 SSG app** — prerendered to
fully static HTML and served by a small Express server on the Mac mini, the **same architecture as
bitsize.me / faunapoolen.se**. English only. **CodeKit is fully retired**; there is no `.kit` build
step and no external "catalyst" dependency (its styles are vendored locally).

## Run

```bash
pnpm install            # Angular 22 + express + @mikaelcedergren/cx-framework
pnpm dev                # ng serve at http://127.0.0.1:4250
pnpm build              # ng build (prerender) -> dist/browser, then scripts/flatten.mjs
pnpm start              # serve dist/browser at http://127.0.0.1:3050 (HOST/PORT env; health: /healthz)
```

Always-on service: `launchd/com.mikaelcedergren.server.plist` (port `3050`, RunAtLoad + KeepAlive),
fronted by nginx. Rebuild + reload after content changes:

```bash
pnpm build && sudo launchctl kickstart -k system/com.mikaelcedergren.server
```

## Architecture

Angular 22 standalone, prerendered to static HTML (`angular.json` -> `outputMode: "static"`; every
route `RenderMode.Prerender`), served as plain files by Express via the shared server library
(`../../server-ops/lib/site-server.mjs`) — no SSR at runtime.

```
src/
  index.html             <head>: favicon, fonts, vendored catalyst.css + styles.css; <mc-root>
  app/
    app.component.*       shared chrome (bottom nav menu + logo), wraps <router-outlet> in <main>
    app.routes.ts         GENERATED by scripts/import-legacy.mjs — route + per-page SEO
    app.routes.server.ts  { path: '**', renderMode: Prerender }
    app.config.ts         router + client hydration + SeoTitleStrategy
    shared/seo.ts         SeoTitleStrategy: per-page title/desc/canonical/OG/Twitter/JSON-LD (single locale)
    pages/**              one component per page; body transplanted verbatim via [innerHTML]
public/assets/**          images, vendored catalyst.css, styles.css — copied to dist/assets
scripts/flatten.mjs       post-build: <route>.html/index.html -> flat <route>.html (blog post URLs)
scripts/import-legacy.mjs ONE-TIME migration importer (archival; expects a separate legacy CodeKit checkout if ever reused)
server/index.mjs          Express static server (shared lib; caching, security headers, /healthz, 404)
```

### URLs (unchanged from the legacy site)

- **Sections** at `<dir>/index.html`: `/about/`, `/concepts/`, `/resume/`, `/blog/`.
- **Blog posts** are literal `.html` files: `/blog/posts/<slug>.html` (Angular prerenders
  `<route>/index.html`, `scripts/flatten.mjs` flattens to a flat file).
- Home at `/`.

## Content source of truth

The **Angular app (`src/`) is the source of truth.** Pages were imported from the legacy CodeKit
output during the migration by `scripts/import-legacy.mjs`, which extracts each page's `<main>` body
and emits a component rendering it via `[innerHTML]` (so arbitrary markup — emails, etc. — survives
the Angular compiler). **Going forward, edit the Angular pages directly** (`src/app/pages/**`);
re-running the importer would require a separate legacy CodeKit checkout and would overwrite pages.

## cx-framework

`@mikaelcedergren/cx-framework` is imported for shared tokens, base styles, and embed styles while
the migrated portfolio keeps its legacy visual skin. Preserve that split: shared foundations from
the framework, site-specific presentation only where the old public look needs it.

This repo sits in the **Cortex -> cx-framework -> projects** loop. Cortex authors reusable
components, tokens, AI skills, guidelines, and framework decisions; `cx-framework` packages them;
this site consumes only `@mikaelcedergren/cx-framework`. Do not reference Cortex directly through
imports, package deps, scripts, styles, local paths, or copied source. If the portfolio reveals a
reusable framework gap, fix it in Cortex, package/push `cx-framework`, then update this repo from the
package.

## Notes / follow-ups

- The legacy site used `simple-parallax-js` (CDN) for scroll parallax; not re-added (no external
  runtime deps). Hero images render statically. Add a vendored equivalent if the effect is wanted.
- Catalyst was an abandoned external CSS framework; its rules are vendored at
  `public/assets/styles/catalyst.css`. Fold what's needed into the site styles (or cx-framework) over time.
